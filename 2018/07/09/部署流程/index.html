项目写好了。有一台服务器。

服务器基本配置

登录

ssh username@ip

这是是root用户，权限太大不安全。

新建用户

新建用户
 adduser name 
给sudo权限
 vim /etc/sudoers
在root ALL=(ALL)ALL这一行下增加
name ALL=(ALL)ALL

简化登录

本机.zshrc文件给命令配置别名

alias tx="ssh name@ip"

无密码登录

和git仓库配置秘钥原理一样。

1. 两边都要有生成秘钥
ssh-keygen -t rsa -C “email”

2. 服务器保存本地公钥
echo pubkey >> .ssh/authorized_keys

3. 重启服务器ssh
service ssh restart

安全措施

1. 修改端口
文件/etc/ssh/sshd_config 
port属性,换成其他端口

2. 关闭root密码登录
PermitRootLogin属性，换成without-password

3. 重启ssh

4. 还可以配置iptables

nginx配置端口代理

在/etc/nginx/nginx.conf中添加

# 让nginx加载配置文件
include /etc/nginx/conf.d/*.conf;

然后在/etc/nginx/conf.d文件夹下新建配置文件name-port.conf

upstream dappdemo {
    server localhost:3000;
}
server {

    listen 80;
    server_name 127.0.0.1;

    location / {
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forward-For $proxy_add_x_forwarded_for;
        proxy_set_header Host $http_host;
        proxy_set_header X-Nginx-Proxy true;
        proxy_pass http://dappdemo;
        proxy_redirect off;
    }
}

匹配到server_name的流量转发到proxy_pass

生成镜像

根据dockerfile新建一个镜像

docker build -t yourname/imagename:tag .

新建容器并启动

用了Makefile

pjName := dappdemo

docker:
    docker run \
        --name ${pjName} \
        --rm \
        -ti \
        -p 3000:3000 \
        benny/dapp:0.1 \
        /bin/bash



make docker
